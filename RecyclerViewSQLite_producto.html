RecyclerView con SQLite: Categorias y Productos

Guia practica desarrollada desde cero para trabajar dos tablas relacionadas en Android Studio

Nivel sugerido: intermedio

Mapa general de la guia

1. Estructura general

Organizar el proyecto con modelos, helper, adapters, activities y layouts desde el inicio.

2. Base de datos relacional

Crear las tablas producto_categoria y producto con relacion por id_categoria.

3. CRUD de categorias

Registrar, listar, editar y eliminar categorias antes de crear productos.

4. CRUD de productos

Registrar productos con nombre, descripcion, precio, stock y categoria elegida en un Spinner.

5. RecyclerView y eventos

Mostrar listas, manejar click, long click, formularios y refresco visual.

6. Mejora recomendada

Agregar busqueda local, mensajes de lista vacia y separacion clara por capas.

PARTE 1: Propuesta general

Enfoque sugerido

La forma mas clara de construir este proyecto es separar las categorias y los productos. Primero se administra producto_categoria en una pantalla propia. Luego se administra producto en la pantalla principal usando un formulario con Spinner para elegir la categoria.

Pantalla 1

CategoriaActivity: lista y CRUD de categorias.

Pantalla 2

MainActivity: lista y CRUD de productos.

Relacion

Cada producto pertenece a una categoria mediante id_categoria.

Secuencia recomendada de trabajo

  1. Crear modelos y helper.
  2. Registrar primero varias categorias.
  3. Despues registrar productos usando el Spinner de categorias.
  4. Listar productos mostrando tambien el nombre de la categoria.

PARTE 2: Estructura del proyecto

Estructura a seguir

app/ ├── manifests/ │ └── AndroidManifest.xml ├── java/com.example.sqlite_producto/ │ ├── adapter/ │ │ ├── ProductoAdapter.kt │ │ └── ProductoCategoriaAdapter.kt │ ├── data/ │ │ └── InventarioDbHelper.kt │ ├── model/ │ │ ├── Producto.kt │ │ ├── ProductoCategoria.kt │ │ └── ProductoItem.kt │ └── view/ │ ├── MainActivity.kt │ └── CategoriaActivity.kt └── res/ └── layout/ ├── activity_main.xml ├── activity_categoria.xml ├── item_producto.xml ├── item_producto_categoria.xml ├── dialog_producto.xml └── dialog_producto_categoria.xml

Por que esta estructura es la mas conveniente

  • ProductoCategoria.kt representa la tabla de categorias.
  • Producto.kt representa la tabla de productos.
  • ProductoItem.kt puede usarse para listar productos junto con el nombre de la categoria.
  • InventarioDbHelper.kt concentra toda la logica SQLite del proyecto.

PARTE 3: Diseno de la base de datos

Tablas necesarias

Se usaran dos tablas. La primera guardara las categorias. La segunda guardara los productos y tendra una referencia a la categoria elegida.

Tabla Campos propuestos Funcion
producto_categoria id, nombre, descripcion Organizar los productos por tipo.
producto id, nombre, descripcion, precio, stock, id_categoria Guardar el producto y su categoria asociada.

Modelo relacional

producto_categoria
→
producto.id_categoria

Codigo completo de InventarioDbHelper.kt

Este helper ya deja listas las dos tablas y una base para el CRUD de categorias y productos.

package com.example.sqlite_producto.data import android.content.ContentValues import android.content.Context import android.database.sqlite.SQLiteDatabase import android.database.sqlite.SQLiteOpenHelper import com.example.sqlite_producto.model.Producto import com.example.sqlite_producto.model.ProductoCategoria import com.example.sqlite_producto.model.ProductoItem class InventarioDbHelper(context: Context) : SQLiteOpenHelper(context, DB_NAME, null, DB_VERSION) { companion object { const val DB_NAME = "inventario.db" const val DB_VERSION = 1 const val TABLE_CATEGORIA = "producto_categoria" const val COL_CAT_ID = "id" const val COL_CAT_NOMBRE = "nombre" const val COL_CAT_DESCRIPCION = "descripcion" const val TABLE_PRODUCTO = "producto" const val COL_PROD_ID = "id" const val COL_PROD_NOMBRE = "nombre" const val COL_PROD_DESCRIPCION = "descripcion" const val COL_PROD_PRECIO = "precio" const val COL_PROD_STOCK = "stock" const val COL_PROD_ID_CATEGORIA = "id_categoria" } override fun onCreate(db: SQLiteDatabase) { val sqlCategoria = """ CREATE TABLE $TABLE_CATEGORIA ( $COL_CAT_ID INTEGER PRIMARY KEY AUTOINCREMENT, $COL_CAT_NOMBRE TEXT NOT NULL, $COL_CAT_DESCRIPCION TEXT NOT NULL ) """.trimIndent() val sqlProducto = """ CREATE TABLE $TABLE_PRODUCTO ( $COL_PROD_ID INTEGER PRIMARY KEY AUTOINCREMENT, $COL_PROD_NOMBRE TEXT NOT NULL, $COL_PROD_DESCRIPCION TEXT NOT NULL, $COL_PROD_PRECIO REAL NOT NULL, $COL_PROD_STOCK INTEGER NOT NULL, $COL_PROD_ID_CATEGORIA INTEGER NOT NULL, FOREIGN KEY($COL_PROD_ID_CATEGORIA) REFERENCES $TABLE_CATEGORIA($COL_CAT_ID) ) """.trimIndent() db.execSQL(sqlCategoria) db.execSQL(sqlProducto) } override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) { db.execSQL("DROP TABLE IF EXISTS $TABLE_PRODUCTO") db.execSQL("DROP TABLE IF EXISTS $TABLE_CATEGORIA") onCreate(db) } fun insertarCategoria(categoria: ProductoCategoria): Long { val db = writableDatabase val values = ContentValues().apply { put(COL_CAT_NOMBRE, categoria.nombre) put(COL_CAT_DESCRIPCION, categoria.descripcion) } val result = db.insert(TABLE_CATEGORIA, null, values) db.close() return result } fun obtenerCategorias(): MutableList<ProductoCategoria> { val lista = mutableListOf<ProductoCategoria>() val db = readableDatabase val cursor = db.rawQuery("SELECT * FROM $TABLE_CATEGORIA ORDER BY $COL_CAT_NOMBRE ASC", null) if (cursor.moveToFirst()) { do { lista.add( ProductoCategoria( id = cursor.getInt(cursor.getColumnIndexOrThrow(COL_CAT_ID)), nombre = cursor.getString(cursor.getColumnIndexOrThrow(COL_CAT_NOMBRE)), descripcion = cursor.getString(cursor.getColumnIndexOrThrow(COL_CAT_DESCRIPCION)) ) ) } while (cursor.moveToNext()) } cursor.close() db.close() return lista } fun actualizarCategoria(categoria: ProductoCategoria): Int { val db = writableDatabase val values = ContentValues().apply { put(COL_CAT_NOMBRE, categoria.nombre) put(COL_CAT_DESCRIPCION, categoria.descripcion) } val result = db.update(TABLE_CATEGORIA, values, "$COL_CAT_ID = ?", arrayOf(categoria.id.toString())) db.close() return result } fun eliminarCategoria(id: Int): Int { val db = writableDatabase val result = db.delete(TABLE_CATEGORIA, "$COL_CAT_ID = ?", arrayOf(id.toString())) db.close() return result } fun insertarProducto(producto: Producto): Long { val db = writableDatabase val values = ContentValues().apply { put(COL_PROD_NOMBRE, producto.nombre) put(COL_PROD_DESCRIPCION, producto.descripcion) put(COL_PROD_PRECIO, producto.precio) put(COL_PROD_STOCK, producto.stock) put(COL_PROD_ID_CATEGORIA, producto.idCategoria) } val result = db.insert(TABLE_PRODUCTO, null, values) db.close() return result } fun obtenerProductosConCategoria(): MutableList<ProductoItem> { val lista = mutableListOf<ProductoItem>() val db = readableDatabase val query = """ SELECT p.id, p.nombre, p.descripcion, p.precio, p.stock, p.id_categoria, c.nombre AS categoria_nombre FROM $TABLE_PRODUCTO p INNER JOIN $TABLE_CATEGORIA c ON p.$COL_PROD_ID_CATEGORIA = c.$COL_CAT_ID ORDER BY p.$COL_PROD_NOMBRE ASC """.trimIndent() val cursor = db.rawQuery(query, null) if (cursor.moveToFirst()) { do { lista.add( ProductoItem( id = cursor.getInt(cursor.getColumnIndexOrThrow("id")), nombre = cursor.getString(cursor.getColumnIndexOrThrow("nombre")), descripcion = cursor.getString(cursor.getColumnIndexOrThrow("descripcion")), precio = cursor.getDouble(cursor.getColumnIndexOrThrow("precio")), stock = cursor.getInt(cursor.getColumnIndexOrThrow("stock")), idCategoria = cursor.getInt(cursor.getColumnIndexOrThrow("id_categoria")), nombreCategoria = cursor.getString(cursor.getColumnIndexOrThrow("categoria_nombre")) ) ) } while (cursor.moveToNext()) } cursor.close() db.close() return lista } fun actualizarProducto(producto: Producto): Int { val db = writableDatabase val values = ContentValues().apply { put(COL_PROD_NOMBRE, producto.nombre) put(COL_PROD_DESCRIPCION, producto.descripcion) put(COL_PROD_PRECIO, producto.precio) put(COL_PROD_STOCK, producto.stock) put(COL_PROD_ID_CATEGORIA, producto.idCategoria) } val result = db.update(TABLE_PRODUCTO, values, "$COL_PROD_ID = ?", arrayOf(producto.id.toString())) db.close() return result } fun eliminarProducto(id: Int): Int { val db = writableDatabase val result = db.delete(TABLE_PRODUCTO, "$COL_PROD_ID = ?", arrayOf(id.toString())) db.close() return result } }

PARTE 6: CRUD de productos con Spinner de categorias

Objetivo de esta parte

El formulario del producto debe permitir elegir una categoria existente. Para eso se cargaran todas las categorias en un Spinner antes de guardar o editar.

Regla importante

Si no existen categorias registradas, no conviene abrir el formulario de productos. Primero se deben crear categorias en CategoriaActivity.

Campos basicos sugeridos para el producto

  • nombre: texto obligatorio.
  • descripcion: texto obligatorio.
  • precio: numero decimal obligatorio.
  • stock: numero entero obligatorio.
  • categoria: seleccion obligatoria desde un Spinner.

PARTE 7: Proceso practico sugerido

Orden de validacion del proyecto

  1. Ejecutar la app y abrir la pantalla de categorias.
  2. Registrar al menos tres categorias.
  3. Volver a la pantalla principal de productos.
  4. Abrir el formulario del producto y comprobar que el Spinner muestre las categorias.
  5. Registrar varios productos con distintas categorias.
  6. Verificar que el RecyclerView muestre tambien el nombre de la categoria.
  7. Probar editar y eliminar tanto categorias como productos.

Puntos delicados del flujo

  • Si el Spinner aparece vacio, revisar primero si existen categorias registradas.
  • Si el producto no muestra categoria, revisar la consulta con INNER JOIN.
  • Si no se actualiza la lista, verificar la llamada a cargarProductos() o cargarCategorias().
  • Si el proyecto no reconoce el FloatingActionButton, confirmar el uso de Material Components.

PARTE 8: Propuesta final

Como se realizaria de forma ordenada

La propuesta mas solida es trabajar con dos pantallas: una para categorias y otra para productos. En la pantalla del producto, el formulario usa un Spinner para elegir la categoria existente. Eso evita duplicar nombres de categorias, mejora la consistencia de los datos y deja el proyecto listo para crecer.

Componente Propuesta
CRUD de categorias En CategoriaActivity, con su propio RecyclerView y FAB.
CRUD de productos En MainActivity, con formulario y Spinner de categorias.
Relacion entre tablas producto.id_categoria apunta a producto_categoria.id.
Lista final Mostrar productos con nombre de categoria usando un JOIN.

Resultado esperado del proyecto

  1. Registrar categorias y verlas en un RecyclerView.
  2. Registrar productos seleccionando una categoria desde un Spinner.
  3. Listar productos con nombre, descripcion, precio, stock y nombre de categoria.
  4. Editar y eliminar categorias y productos desde eventos visuales.
