📚 PARTE 1: ¿Qué es RecyclerView?

Definición

RecyclerView es un componente avanzado de Android para mostrar grandes conjuntos de datos de manera eficiente. Es la evolución moderna de ListView, diseñado para optimizar el rendimiento mediante el reciclaje de vistas.

🔄 Concepto de Reciclaje

❌ ListView tradicional:

Crea una vista nueva para CADA item → 1000 items = 1000 vistas

✅ RecyclerView:

Crea solo las vistas visibles en pantalla (~10 vistas)

Cuando haces scroll, REUTILIZA esas vistas con nuevos datos

1000 items = Solo 10-15 vistas creadas ♻️

✅ Ventajas de RecyclerView

  • 🚀 Alto rendimiento: Recicla vistas eficientemente
  • 💾 Menor consumo de memoria: Solo carga vistas visibles
  • 🎨 Múltiples layouts: Linear, Grid, Staggered
  • ✨ Animaciones integradas: Add, remove, move items
  • 🔧 Altamente personalizable: ItemDecorator, ItemAnimator
  • 🔄 ViewHolder obligatorio: Mejor organización del código

❌ Desventajas / Consideraciones

  • 📚 Más complejo: Requiere más código inicial
  • ⚙️ Configuración manual: Necesitas LayoutManager
  • 🎓 Curva de aprendizaje: Conceptos de Adapter y ViewHolder
💡 ¿Cuándo usar RecyclerView?
Situación ListView RecyclerView
Lista pequeña (<50 items) ✅ OK ✅ Mejor
Lista grande (>100 items) ❌ Lento ✅✅ Excelente
Necesitas Grid/Horizontal ❌ Complicado ✅✅ Fácil
Animaciones ❌ Manual ✅✅ Integradas
Proyecto nuevo ❌ Legacy ✅✅ Recomendado
⚠️ Importante:

Google recomienda usar RecyclerView en lugar de ListView para todos los proyectos nuevos. ListView se considera legacy (obsoleto).

🏗️ PARTE 2: Arquitectura de RecyclerView

Componentes Principales

1️⃣ RecyclerView

El contenedor principal que muestra los items en pantalla

2️⃣ Adapter (Adaptador)

Intermediario entre los datos y el RecyclerView

Responsabilidades:

  • Crear ViewHolders cuando sea necesario
  • Vincular datos con ViewHolders
  • Indicar cuántos items hay

3️⃣ ViewHolder

Contenedor que guarda las referencias a las vistas de cada item

Ventaja: Evita llamadas repetidas a findViewById()

4️⃣ LayoutManager

Define cómo se organizan los items

Tipos:

  • LinearLayoutManager: Lista vertical u horizontal
  • GridLayoutManager: Cuadrícula (como galería)
  • StaggeredGridLayoutManager: Cuadrícula irregular (Pinterest)

5️⃣ ItemDecoration (Opcional)

Agrega decoraciones como separadores, márgenes, etc.

6️⃣ ItemAnimator (Opcional)

Controla las animaciones al agregar/eliminar/mover items

🔄 Ciclo de Vida del RecyclerView

1. onCreateViewHolder()

Se llama cuando RecyclerView necesita crear un nuevo ViewHolder. Aquí inflamos el layout del item.

2. onBindViewHolder()

Se llama para vincular datos a un ViewHolder existente. Aquí asignamos los datos a las vistas.

3. getItemCount()

Retorna la cantidad total de items en la lista.

💡 ViewHolder Pattern:

El patrón ViewHolder es OBLIGATORIO en RecyclerView. Guarda las referencias a las vistas para reutilizarlas, evitando múltiples llamadas costosas a findViewById().

❌ Sin ViewHolder (ListView antiguo):

findViewById() se llama cada vez que se muestra un item

1000 scrolls = 1000 llamadas a findViewById() 🐌

✅ Con ViewHolder (RecyclerView):

findViewById() solo se llama al crear el ViewHolder

1000 scrolls = Solo ~10 llamadas iniciales 🚀

PASO 5

ViewHolder — adapter/UsuarioViewHolder.kt

Guarda las referencias a las vistas del XML. Al hacerlo una sola vez por ítem se evita llamar a findViewById() repetidamente, lo que mejora el rendimiento.

📄 adapter/UsuarioViewHolder.kt

package com.example.reciclerviewbb.adapter

            import android.view.View
            import android.widget.TextView
            import androidx.recyclerview.widget.RecyclerView
            import com.example.reciclerviewbb.R

            // ====================================================
            // COMPONENTE 3: VIEWHOLDER
            // ====================================================
            // Guarda las referencias a las vistas de cada ítem.
            // Así evitamos llamar a findViewById muchas veces (mejora el rendimiento).
            class UsuarioViewHolder(view: View) : RecyclerView.ViewHolder(view) {

            // Referencias a los TextView del layout item_usuario.xml
            val tvNombre: TextView = view.findViewById(R.id.tvNombre)
            val tvCorreo: TextView = view.findViewById(R.id.tvCorreo)
            }
PASO 6

Adapter — adapter/UsuarioAdapter.kt

Es el puente entre los datos y la vista. Implementa tres métodos clave: onCreateViewHolder, onBindViewHolder y getItemCount.

📄 adapter/UsuarioAdapter.kt

package com.example.reciclerviewbb.adapter

            import android.view.LayoutInflater
            import android.view.ViewGroup
            import android.widget.Toast
            import androidx.recyclerview.widget.RecyclerView
            import com.example.reciclerviewbb.R
            import com.example.reciclerviewbb.model.Usuario

            // ====================================================
            // COMPONENTE 2: ADAPTER
            // ====================================================
            // Conecta los datos (lista de usuarios) con el RecyclerView.
            class UsuarioAdapter(private val listaUsuarios: List<Usuario>) :
            RecyclerView.Adapter<UsuarioViewHolder>() {

            // 1. CREA un nuevo ViewHolder cuando el RecyclerView lo necesita
            // Aquí "inflamos" (convertimos XML a vista) el layout item_usuario.xml
            override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): UsuarioViewHolder {
            val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_usuario, parent, false)
            return UsuarioViewHolder(view)
            }

            // 2. LLENA el ViewHolder con los datos según la posición
            override fun onBindViewHolder(holder: UsuarioViewHolder, position: Int) {
            val usuario = listaUsuarios[position]

            holder.tvNombre.text = usuario.nombre
            holder.tvCorreo.text = usuario.correo

            // Detectar clic en cada ítem de la lista
            holder.itemView.setOnClickListener {
            Toast.makeText(
            holder.itemView.context,
            "Tocaste a: ${usuario.nombre}",
            Toast.LENGTH_SHORT
            ).show()
            }
            }

            // 3. Devuelve la CANTIDAD total de ítems en la lista
            override fun getItemCount(): Int = listaUsuarios.size
            }

🗺️ Guía Paso a Paso

Implementación de RecyclerView en Android

⚙️ Fase 1: Configuración del Entorno (Solución de Errores)

Antes de programar, es vital que el "motor" del proyecto (Gradle) esté bien configurado. En este proyecto se corrigieron los siguientes puntos:

  1. Versión de Java: Se cambió de Java 11 a Java 17. Las librerías modernas de Android ya no son compatibles con Java 11.
  2. SDK: Se ajustó el compileSdk y targetSdk a 34 o 35 (versiones estables), ya que la 36 es experimental.
  3. Dependencias: Se corrigió core-ktx a la versión 1.13.1, ya que la 1.18.0 causaba que la app se cerrara inesperadamente (SIG: 9).

🗂️ Fase 2: El Modelo de Datos (La Clase Usuario)

Antes de mostrar datos, es necesario definir qué datos se van a mostrar.

  • Archivo: model/Usuario.kt
  • Concepto: Una data class es una forma concisa de crear objetos que solo contienen información.

Kotlin

data class Usuario(val nombre: String, val correo: String)

🎨 Fase 3: El Diseño Visual (XML)

Un RecyclerView requiere dos archivos de diseño:

  1. El contenedor principal: Donde vive el RecyclerView completo (activity_main.xml).
  2. El diseño del ítem: Define cómo se verá una sola fila de la lista (item_usuario.xml).
    • Se utiliza un TextView para el nombre (tvNombre) y otro para el correo (tvCorreo).

🧠 Fase 4: El Adaptador y el ViewHolder (El "Cerebro")

El Adapter es el puente entre los datos y la vista. Es la parte central de la arquitectura.

  • ViewHolder: Su trabajo es "encontrar" las vistas del XML (findViewById) una sola vez para ahorrar memoria.
  • Funciones principales del Adapter:
    1. onCreateViewHolder: Infla (crea) el diseño del ítem.
    2. onBindViewHolder: Asigna los datos del usuario (nombre/correo) en los textos correspondientes.
    3. getItemCount: Indica cuántos elementos tiene la lista.

🔗 Fase 5: La Lógica en MainActivity

Aquí es donde se une todo. Los pasos clave son:

  1. Inicializar la vista: Usar findViewById para conectar con el RecyclerView del XML.
  2. Preparar los datos: Crear una lista (listOf) de objetos Usuario.
  3. ⚠️ Configurar el LayoutManager (Punto Crítico): Sin un LayoutManager (como LinearLayoutManager), la lista no sabe si mostrarse vertical u horizontalmente y la app falla.
  4. Asignar el Adaptador: Indicarle al RecyclerView quién va a gestionar sus datos.

🎵 Fase 6: Recursos Multimedia (Carpeta raw)

Para gestionar recursos externos como audio o video en el proyecto:

  1. Crear el directorio res/raw.
  2. Seguir las reglas de nombres: todo en minúsculas, sin espacios y sin números al inicio.
  3. Uso de la clase R: Es el índice automático de Android. Al colocar un archivo musica.mp3, Android genera automáticamente R.raw.musica.

📋 Fase 7: El Manifiesto (AndroidManifest.xml)

Este archivo es el "DNI" de la aplicación. Define su identidad y estructura ante el sistema operativo.

  • En este archivo se define qué actividad arranca primero mediante el <intent-filter> con la acción MAIN y categoría LAUNCHER.
  • La MainActivity se ubica dentro del paquete .view.

package com.example.ucv_b

            import android.view.LayoutInflater
            import android.view.View
            import android.view.ViewGroup
            import android.widget.TextView
            import androidx.recyclerview.widget.RecyclerView

            class FrutaAdapter(
            private val frutas: List<Fruta>
            ) : RecyclerView.Adapter<FrutaAdapter.FrutaViewHolder>() {

            // ViewHolder: Guarda referencias a las vistas
            class FrutaViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
            val tvEmoji: TextView = itemView.findViewById(R.id.tvEmoji)
            val tvNombre: TextView = itemView.findViewById(R.id.tvNombre)
            }

            // 1. Crear el ViewHolder (inflar el layout)
            override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): FrutaViewHolder {
            val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_fruta, parent, false)
            return FrutaViewHolder(view)
            }

            // 2. Vincular datos con el ViewHolder
            override fun onBindViewHolder(holder: FrutaViewHolder, position: Int) {
            val fruta = frutas[position]
            holder.tvEmoji.text = fruta.emoji
            holder.tvNombre.text = fruta.nombre
            }

            // 3. Retornar cantidad de items
            override fun getItemCount(): Int = frutas.size
            }

📖 Explicación Detallada del Adapter

1️⃣ Declaración de la clase

  • Recibe la lista de frutas como parámetro
  • Hereda de RecyclerView.Adapter (clase base obligatoria)
  • Especifica que usará FrutaViewHolder como contenedor de vistas

🔍 Desglosando List<Fruta>:

1. List - Interfaz de Kotlin

  • Es una interfaz built-in (incorporada) de Kotlin
  • Viene incluida en el lenguaje, no necesitas declararla
  • Representa una lista inmutable (solo lectura)

2. <Fruta> - Tipo genérico

  • Los < > indican genéricos (qué tipo de objetos contiene la lista)
  • Fruta es tu clase que se declara en Fruta.kt
  • Significa: "Esta es una lista de objetos tipo Fruta"
// Ejemplos de genéricos:
List<String> // Lista de Strings
List<Int> // Lista de enteros
List<Fruta> // Lista de Frutas (tu clase)
List<Persona> // Lista de Personas (si tuvieras esa clase)

2️⃣ FrutaViewHolder (Clase interna)

  • Función: Guarda las referencias a los TextView del layout
  • Por qué: Evita hacer findViewById repetidamente (más eficiente)
  • Analogía: Es como marcar con post-its los lugares importantes en un libro para no buscarlos cada vez

3️⃣ onCreateViewHolder() - CREA la vista

  • Cuándo se llama: Solo cuando se necesita crear una nueva vista (12-15 veces)
  • Qué hace: Infla el layout item_fruta.xml y crea un ViewHolder
  • Analogía: Construye el molde físico de la galleta

4️⃣ onBindViewHolder() - RELLENA con datos

  • Cuándo se llama: Cada vez que un item se va a mostrar (miles de veces al hacer scroll)
  • Qué hace: Toma la fruta en la posición indicada y asigna sus datos a los TextView
  • Analogía: Rellena el molde con la masa (datos específicos)

5️⃣ getItemCount() - CUENTA total

  • Qué hace: Devuelve cuántos items hay en total (5 en tu caso, o 5000 si tuvieras más)
  • Para qué: RecyclerView sabe cuándo dejar de hacer scroll

🔄 Proceso Completo

  1. getItemCount() → "Tengo 5 frutas"
  2. onCreateViewHolder() → "Creo 12 vistas físicas" (solo al inicio)
  3. onBindViewHolder(position=0) → "Relleno vista 1 con 🍎 Manzana"
  4. onBindViewHolder(position=1) → "Relleno vista 2 con 🍌 Plátano"
  5. [Usuario hace scroll]
  6. onBindViewHolder(position=5) → "Reciclo vista 1, ahora tiene otros datos"

🎨 LayoutManagers

Tipos de LayoutManager

El LayoutManager define cómo se organizan los items en el RecyclerView

1️⃣ LinearLayoutManager (Lista Vertical u Horizontal)

📱 Lista Vertical (Por defecto)

// Lista vertical normal
            recyclerView.layoutManager = LinearLayoutManager(this)

📱 Lista Horizontal

// Lista horizontal (como carrusel)
            recyclerView.layoutManager = LinearLayoutManager(
            this,
            LinearLayoutManager.HORIZONTAL,
            false
            )

📱 Lista Invertida (de abajo hacia arriba)

// Lista invertida (útil para chats)
            recyclerView.layoutManager = LinearLayoutManager(
            this,
            LinearLayoutManager.VERTICAL,
            true // reverseLayout
            )

2️⃣ GridLayoutManager (Cuadrícula)

📱 Grid con 2 columnas

// Cuadrícula de 2 columnas (galería)
            recyclerView.layoutManager = GridLayoutManager(this, 2)

📱 Grid con 3 columnas

// Cuadrícula de 3 columnas
            recyclerView.layoutManager = GridLayoutManager(this, 3)

📱 Grid Horizontal

// Cuadrícula horizontal
            recyclerView.layoutManager = GridLayoutManager(
            this,
            2, // filas
            GridLayoutManager.HORIZONTAL,
            false
            )

3️⃣ StaggeredGridLayoutManager (Grid Irregular)

📱 Estilo Pinterest

// Grid irregular (como Pinterest)

            2, // columnas
            StaggeredGridLayoutManager.VERTICAL
            )

⚡ Eventos Click en Items

⚠️ Diferencia importante:

RecyclerView NO tiene setOnItemClickListener() como ListView. Debes implementar el click manualmente.

💻 Método 1: Click en el Adapter (Recomendado)

📱 FrutaAdapter.kt (Con Click)

class FrutaAdapter(
            private val frutas: List<Fruta>,
            private val onItemClick: (Fruta) -> Unit // ⭐ Lambda para click
            ) : RecyclerView.Adapter<FrutaAdapter.FrutaViewHolder>() {

            class FrutaViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) {
            val tvEmoji: TextView = itemView.findViewById(R.id.tvEmoji)
            val tvNombre: TextView = itemView.findViewById(R.id.tvNombre)
            }

            override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): FrutaViewHolder {
            val view = LayoutInflater.from(parent.context)
            .inflate(R.layout.item_fruta, parent, false)
            return FrutaViewHolder(view)
            }

            override fun onBindViewHolder(holder: FrutaViewHolder, position: Int) {
            val fruta = frutas[position]
            holder.tvEmoji.text = fruta.emoji
            holder.tvNombre.text = fruta.nombre

            // ⭐ Configurar click en el item completo
            holder.itemView.setOnClickListener {
            onItemClick(fruta)
            }
            }

            override fun getItemCount(): Int = frutas.size
            }

Operaciones CRUD en RecyclerView

A diferencia de ListView, RecyclerView requiere usar ArrayList (mutable) y notificar cambios específicos.

class FrutaAdapter(

💡 Métodos de notificación importantes:
Método Uso Animación
notifyDataSetChanged() Actualizar todo ❌ No
notifyItemInserted(position) Item agregado ✅ Sí
notifyItemRemoved(position) Item eliminado ✅ Sí
notifyItemChanged(position) Item modificado ✅ Sí
notifyItemMoved(from, to) Item movido ✅ Sí
⚠️ Importante:

Usa notifyItemInserted(), notifyItemRemoved(), etc. en lugar de notifyDataSetChanged() para obtener animaciones automáticas y mejor rendimiento.

class FrutaAdapter(
            private var frutas: ArrayList<Fruta>
            ) : RecyclerView.Adapter<FrutaAdapter.FrutaViewHolder>() {

            private var frutasFiltradas = ArrayList<Fruta>(frutas)

            // ... ViewHolder y métodos anteriores ...

            fun filtrar(texto: String) {
            frutasFiltradas.clear()

            if (texto.isEmpty()) {
            frutasFiltradas.addAll(frutas)
            } else {
            val busqueda = texto.lowercase()
            for (fruta in frutas) {
            if (fruta.nombre.lowercase().contains(busqueda)) {
            frutasFiltradas.add(fruta)
            }
            }
            }

            notifyDataSetChanged()
            }

            override fun getItemCount(): Int = frutasFiltradas.size
            }
