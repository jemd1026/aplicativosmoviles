Aplicación de Reproductor de Música

Vamos a entender el ciclo de vida de Android usando una app de música.

📱 Escenario: App de Reproductor de Música

Imagina que estás desarrollando MusicApp, un reproductor de música simple. El usuario puede reproducir canciones, pausarlas, y la música debe seguir sonando incluso cuando el usuario sale de la app para responder un mensaje de WhatsApp.

🎵

MusicApp

Reproduciendo: "Canción Favorita"

🎵 Selecciona una canción

▶️ Música comienza a reproducirse
⬇️

⬇️

⬇️

🔄 ¿Qué Hace Cada Método del Ciclo de Vida?

🎬
Paso 1: Crear
onCreate()
¿Cuándo? Usuario abre la app por primera vez.
✅ Cargar la lista de canciones
✅ Inicializar el reproductor de música
✅ Configurar los botones (Play, Pause, Next)
👁️
Paso 2: Mostrar
onStart()
¿Cuándo? La app va a ser visible en pantalla.
✅ Actualizar la interfaz
✅ Mostrar la canción actual
✅ Preparar los controles
▶️
Paso 3: Activar
onResume()
¿Cuándo? El usuario puede interactuar con la app.
✅ Reproducir la música
✅ Activar botones de control
✅ Animar la visualización de audio
⏸️
Paso 4: Pausar
onPause()
¿Cuándo? Usuario abre otra app (WhatsApp).
⚠️ NO pausar la música aquí
✅ Detener animaciones visuales
✅ Guardar el tiempo de reproducción
⏹️
Paso 5: Detener
onStop()
¿Cuándo? La app ya no es visible.
✅ Continuar música en segundo plano
✅ Liberar recursos visuales
✅ Guardar el estado de reproducción
💥
Paso 6: Destruir
onDestroy()
¿Cuándo? Usuario cierra la app completamente.
✅ Detener la música
✅ Liberar el reproductor
✅ Cerrar conexiones de audio

💻 Código Completo del Ejemplo

package com.example.ucv2026a import android.media.MediaPlayer import android.os.Bundle import android.util.Log import android.widget.Button import android.widget.TextView import androidx.activity.enableEdgeToEdge import androidx.appcompat.app.AppCompatActivity import androidx.core.view.ViewCompat import androidx.core.view.WindowInsetsCompat class MainActivity : AppCompatActivity() { private var mediaPlayer: MediaPlayer? = null private lateinit var btnPlay: Button private lateinit var btnPause: Button private lateinit var tvSongName: TextView override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) enableEdgeToEdge() // ASEGÚRATE de que este layout exista en res/layout/activity_main.xml setContentView(R.layout.activity_main) ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets -> val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars()) v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom) insets } // Inicializar componentes (Asegúrate de que estos IDs existan en tu XML) btnPlay = findViewById(R.id.btnPlay) btnPause = findViewById(R.id.btnPause) tvSongName = findViewById(R.id.tvSongName) // Crear el reproductor (Asegúrate de que el archivo existe en res/raw/juaneco.mp3) // Si no tienes la carpeta raw, créala: res -> New -> Android Resource Directory (tipo raw) try { mediaPlayer = MediaPlayer.create(this, R.raw.juaneco) if (mediaPlayer == null) { Log.e("MusicApp", "❌ ERROR: No se pudo crear el MediaPlayer. Revisa que R.raw.juaneco exista.") } else { Log.d("MusicApp", "✅ MediaPlayer creado con éxito") } } catch (e: Exception) { Log.e("MusicApp", "❌ Error al cargar el audio: ${e.message}") } btnPlay.setOnClickListener { Log.d("MusicApp", "Botón Play presionado") if (mediaPlayer != null) { mediaPlayer?.start() tvSongName.text = "🎵 Reproduciendo..." } else { Log.e("MusicApp", "El MediaPlayer es nulo, no puede sonar") } } btnPause.setOnClickListener { Log.d("MusicApp", "Botón Pausa presionado") mediaPlayer?.pause() tvSongName.text = "⏸️ Pausado" } Log.d("MusicApp", "🎬 onCreate: App lista") } override fun onStart() { super.onStart() tvSongName.text = "🎵 Reproduciendo: Canción Favorita" Log.d("MusicApp", "👁️ onStart: App visible") } override fun onResume() { super.onResume() Log.d("MusicApp", "▶️ onResume: Interactuando") } override fun onPause() { super.onPause() Log.d("MusicApp", "⏸️ onPause: En segundo plano") } override fun onStop() { super.onStop() Log.d("MusicApp", "⏹️ onStop: Oculta") } override fun onDestroy() { super.onDestroy() mediaPlayer?.release() mediaPlayer = null Log.d("MusicApp", "💥 onDestroy: Destruida") } }

🧩 Estructura Completa de la Pantalla

Para construir esta pantalla en Android se necesitan varias partes trabajando en conjunto: la Activity en Kotlin, la vista XML, el recurso de audio y su correcta vinculación dentro del proyecto.

📄 Archivo Kotlin

Aquí se programa la lógica del ciclo de vida, los botones, enableEdgeToEdge() y el uso de MediaPlayer. En este caso: MainActivity.kt.

🖼️ Archivo de Vista

La interfaz visual se diseña en activity_main.xml. Allí estarán el contenedor con id main, el texto principal y los botones btnPlay y btnPause.

🎵 Recurso de Audio

El archivo de música debe colocarse en res/raw/ con el nombre juaneco.mp3 para poder usarlo con R.raw.juaneco.

🖼️ Vista XML de la Pantalla

<?xml version="1.0" encoding="utf-8"?> <!-- El ID "main" es necesario para ViewCompat.setOnApplyWindowInsetsListener --> <androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android" xmlns:app="http://schemas.android.com/apk/res-auto" xmlns:tools="http://schemas.android.com/tools" android:id="@+id/main" android:layout_width="match_parent" android:layout_height="match_parent" android:padding="20dp" tools:context=".MainActivity"> <TextView android:id="@+id/tvSongName" android:layout_width="wrap_content" android:layout_height="wrap_content" android:text="Presiona Play para escuchar" android:textSize="20sp" android:textStyle="bold" app:layout_constraintBottom_toTopOf="@+id/btnPlay" app:layout_constraintEnd_toEndOf="parent" app:layout_constraintStart_toStartOf="parent" app:layout_constraintTop_toTopOf="parent" app:layout_constraintVertical_chainStyle="packed" /> <Button android:id="@+id/btnPlay" android:layout_width="wrap_content" android:layout_height="wrap_content" android:layout_marginTop="32dp" android:text="REPRODUCIR" app:layout_constraintBottom_toTopOf="@+id/btnPause" app:layout_constraintEnd_toEndOf="parent" app:layout_constraintStart_toStartOf="parent" app:layout_constraintTop_toBottomOf="@+id/tvSongName" /> <Button android:id="@+id/btnPause" android:layout_width="wrap_content" android:layout_height="wrap_content" android:layout_marginTop="16dp" android:text="PAUSAR" app:layout_constraintBottom_toBottomOf="parent" app:layout_constraintEnd_toEndOf="parent" app:layout_constraintStart_toStartOf="parent" app:layout_constraintTop_toBottomOf="@+id/btnPlay" /> </androidx.constraintlayout.widget.ConstraintLayout>
📌 Importante: Los identificadores main, tvSongName, btnPlay y btnPause deben ser exactamente los mismos en el XML y en el código Kotlin. Si no coinciden, Android Studio marcará error.

📦 Recurso y Registro Básico

// Colocar el archivo de audio en: res/raw/juaneco.mp3 // Si esta pantalla no es la principal, registrarla en AndroidManifest.xml <activity android:name=".MainActivity" />
🔧 Si R.raw.juaneco aparece en rojo: haz clic derecho en la carpeta res, luego ve a New > Android Resource Directory, elige raw en Resource type y pulsa OK. Después copia tu audio dentro de res/raw/ con un nombre en minúsculas y sin espacios, por ejemplo juaneco.mp3. Así Android podrá reconocerlo como R.raw.juaneco.
✅ Idea clave: En Android una pantalla completa casi siempre se construye con dos partes: la lógica en Kotlin y la vista en XML. De esta manera se define qué hace la aplicación y cómo se muestra en pantalla.

📊 Tabla Comparativa de Acciones

Método ¿Qué hace el usuario? ¿Qué hace la app? ¿La música suena?
onCreate() Abre la app Carga la canción y prepara botones ❌ No
onStart() Ve la pantalla Muestra la interfaz ❌ No
onResume() Presiona Play Reproduce la música ✅ Sí
onPause() Abre WhatsApp Detiene animaciones ✅ Sí (continúa)
onStop() Usa WhatsApp Libera recursos visuales ✅ Sí (en segundo plano)
onRestart() Vuelve a la app Restaura la interfaz ✅ Sí
onDestroy() Cierra la app Detiene y libera todo ❌ No
💡 Tip #1: En onPause() NO detengas la música si quieres que siga sonando cuando el usuario cambie de app.
💡 Tip #2: Siempre libera recursos pesados (como MediaPlayer) en onDestroy() para evitar fugas de memoria.
💡 Tip #3: Usa Log.d() para ver en LogCat cuándo se ejecuta cada método mientras pruebas tu app.

🎯 Ejercicio Práctico para Ti

Reto: Modifica el código para que la música se pause automáticamente cuando el usuario reciba una llamada telefónica (en onPause()) pero continúe sonando cuando solo cambie de app.

🎓 Resumen del Ejemplo

onCreate: Preparar todo 🎬
onStart: Mostrar interfaz 👁️
onResume: Activar interacción ▶️
onPause: Usuario cambia de app ⏸️
onStop: App oculta ⏹️
onRestart: Usuario regresa 🔄
onDestroy: Limpiar todo 💥
