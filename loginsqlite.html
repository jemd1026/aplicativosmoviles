Login con SQLite en Android

Guia practica desarrollada para registro, acceso local y dashboard basico usando SQLite

Nivel sugerido: intermedio

Mapa general de la guia

1. Estructura general

Organizar pantallas, modelos, helper y layouts para registro, login y dashboard.

2. Base de datos

Crear la tabla usuario con limites maximos y validaciones.

3. Registro

Guardar nombres, DNI, direccion, email, usuario y clave en SQLite.

4. Login

Ingresar con usuario o correo y validar la clave desde SQLite.

5. Dashboard

Mostrar una pantalla sencilla con gadgets de compras, ventas, productos y saldo.

6. Flujo completo

Si no existe cuenta, se registra; si existe, se inicia sesion y se entra al dashboard.

PARTE 1: Propuesta general

Enfoque sugerido

La aplicacion debe iniciar en una pantalla de login moderna, con logo, dos campos y un acceso a registro. Si la persona aun no tiene credenciales, se abre la pantalla de registro. Una vez validado el acceso, se muestra un dashboard sencillo con informacion estatica para practicar la navegacion despues del login.

Pantalla 1

LoginActivity: ingreso con usuario o correo y clave.

Pantalla 2

RegistroActivity: alta del usuario si aun no tiene cuenta.

Pantalla 3

DashboardActivity: vista posterior al acceso, con gadgets estaticos.

Secuencia de trabajo recomendada

  1. Crear la tabla usuario en SQLite.
  2. Construir primero el registro.
  3. Luego construir el login usando esa misma tabla.
  4. Finalmente mostrar el dashboard al validar las credenciales.

PARTE 2: Estructura del proyecto

Estructura a seguir

app/ ├── manifests/ │ └── AndroidManifest.xml ├── java/com.example.login_sqlite/ │ ├── data/ │ │ └── UsuarioDbHelper.kt │ ├── model/ │ │ └── Usuario.kt │ └── view/ │ ├── LoginActivity.kt │ ├── RegistroActivity.kt │ └── DashboardActivity.kt └── res/ └── layout/ ├── activity_login.xml ├── activity_registro.xml └── activity_dashboard.xml

Por que esta estructura es conveniente

  • Usuario.kt representa una fila de la tabla usuario.
  • UsuarioDbHelper.kt concentra toda la logica SQLite de registro y acceso.
  • LoginActivity y RegistroActivity quedan separadas para no mezclar procesos.
  • DashboardActivity solo se preocupa por la vista posterior al login.

PARTE 3: Diseno de la tabla usuario

Campos necesarios

La tabla usuario tendra los siguientes campos: nombres, dni, direccion, email, usuario y clave. Como se desea controlar el tamano maximo de cada campo, conviene usar restricciones CHECK en la sentencia SQL y validaciones previas desde la interfaz.

Campo Tipo Regla propuesta
nombres TEXT Obligatorio, maximo 80 caracteres
dni TEXT Obligatorio, exactamente 8 digitos
direccion TEXT Obligatoria, maximo 120 caracteres
email TEXT Obligatorio, unico, maximo 100 caracteres
usuario TEXT Obligatorio, unico, maximo 30 caracteres
clave TEXT Obligatoria, minimo 6 y maximo 20 caracteres

Modelo Usuario.kt

package com.example.login_sqlite.model data class Usuario( val id: Int = 0, val nombres: String, val dni: String, val direccion: String, val email: String, val usuario: String, val clave: String )

Helper completo: UsuarioDbHelper.kt

Este helper crea la tabla, registra usuarios, verifica si existen credenciales repetidas y permite iniciar sesion con usuario o correo.

package com.example.login_sqlite.data import android.content.ContentValues import android.content.Context import android.database.sqlite.SQLiteDatabase import android.database.sqlite.SQLiteOpenHelper import com.example.login_sqlite.model.Usuario class UsuarioDbHelper(context: Context) : SQLiteOpenHelper(context, DB_NAME, null, DB_VERSION) { companion object { const val DB_NAME = "login_app.db" const val DB_VERSION = 1 const val TABLE_USUARIO = "usuario" const val COL_ID = "id" const val COL_NOMBRES = "nombres" const val COL_DNI = "dni" const val COL_DIRECCION = "direccion" const val COL_EMAIL = "email" const val COL_USUARIO = "usuario" const val COL_CLAVE = "clave" } override fun onCreate(db: SQLiteDatabase) { val sqlUsuario = """ CREATE TABLE $TABLE_USUARIO ( $COL_ID INTEGER PRIMARY KEY AUTOINCREMENT, $COL_NOMBRES TEXT NOT NULL CHECK(LENGTH($COL_NOMBRES) <= 80), $COL_DNI TEXT NOT NULL UNIQUE CHECK(LENGTH($COL_DNI) = 8), $COL_DIRECCION TEXT NOT NULL CHECK(LENGTH($COL_DIRECCION) <= 120), $COL_EMAIL TEXT NOT NULL UNIQUE CHECK(LENGTH($COL_EMAIL) <= 100), $COL_USUARIO TEXT NOT NULL UNIQUE CHECK(LENGTH($COL_USUARIO) <= 30), $COL_CLAVE TEXT NOT NULL CHECK(LENGTH($COL_CLAVE) >= 6 AND LENGTH($COL_CLAVE) <= 20) ) """.trimIndent() db.execSQL(sqlUsuario) } override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) { db.execSQL("DROP TABLE IF EXISTS $TABLE_USUARIO") onCreate(db) } fun registrarUsuario(usuario: Usuario): Long { val db = writableDatabase val values = ContentValues().apply { put(COL_NOMBRES, usuario.nombres) put(COL_DNI, usuario.dni) put(COL_DIRECCION, usuario.direccion) put(COL_EMAIL, usuario.email) put(COL_USUARIO, usuario.usuario) put(COL_CLAVE, usuario.clave) } val result = db.insert(TABLE_USUARIO, null, values) db.close() return result } fun existeUsuarioOEmail(usuario: String, email: String, dni: String): Boolean { val db = readableDatabase val query = """ SELECT * FROM $TABLE_USUARIO WHERE $COL_USUARIO = ? OR $COL_EMAIL = ? OR $COL_DNI = ? """.trimIndent() val cursor = db.rawQuery(query, arrayOf(usuario, email, dni)) val existe = cursor.count > 0 cursor.close() db.close() return existe } fun loginConUsuarioOCorreo(usuarioOEmail: String, clave: String): Usuario? { val db = readableDatabase val query = """ SELECT * FROM $TABLE_USUARIO WHERE ($COL_USUARIO = ? OR $COL_EMAIL = ?) AND $COL_CLAVE = ? """.trimIndent() val cursor = db.rawQuery(query, arrayOf(usuarioOEmail, usuarioOEmail, clave)) var usuarioEncontrado: Usuario? = null if (cursor.moveToFirst()) { usuarioEncontrado = Usuario( id = cursor.getInt(cursor.getColumnIndexOrThrow(COL_ID)), nombres = cursor.getString(cursor.getColumnIndexOrThrow(COL_NOMBRES)), dni = cursor.getString(cursor.getColumnIndexOrThrow(COL_DNI)), direccion = cursor.getString(cursor.getColumnIndexOrThrow(COL_DIRECCION)), email = cursor.getString(cursor.getColumnIndexOrThrow(COL_EMAIL)), usuario = cursor.getString(cursor.getColumnIndexOrThrow(COL_USUARIO)), clave = cursor.getString(cursor.getColumnIndexOrThrow(COL_CLAVE)) ) } cursor.close() db.close() return usuarioEncontrado } }

Proceso de logueo

  1. La persona escribe usuario o correo.
  2. Luego escribe su clave.
  3. Se valida que ambos campos no esten vacios.
  4. Se consulta SQLite con loginConUsuarioOCorreo().
  5. Si existe coincidencia, se abre el dashboard.
  6. Si no existe coincidencia, se muestra un mensaje de error.

Codigo completo: LoginActivity.kt

package com.example.login_sqlite.view import android.content.Intent import android.os.Bundle import android.widget.Button import android.widget.EditText import android.widget.TextView import android.widget.Toast import androidx.appcompat.app.AppCompatActivity import com.example.login_sqlite.R import com.example.login_sqlite.data.UsuarioDbHelper class LoginActivity : AppCompatActivity() { private lateinit var etUsuarioOCorreo: EditText private lateinit var etClaveLogin: EditText private lateinit var btnIngresar: Button private lateinit var tvCrearCuenta: TextView private lateinit var dbHelper: UsuarioDbHelper override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) setContentView(R.layout.activity_login) etUsuarioOCorreo = findViewById(R.id.etUsuarioOCorreo) etClaveLogin = findViewById(R.id.etClaveLogin) btnIngresar = findViewById(R.id.btnIngresar) tvCrearCuenta = findViewById(R.id.tvCrearCuenta) dbHelper = UsuarioDbHelper(this) btnIngresar.setOnClickListener { iniciarSesion() } tvCrearCuenta.setOnClickListener { startActivity(Intent(this, RegistroActivity::class.java)) } } private fun iniciarSesion() { val usuarioOEmail = etUsuarioOCorreo.text.toString().trim() val clave = etClaveLogin.text.toString().trim() if (usuarioOEmail.isEmpty() || clave.isEmpty()) { Toast.makeText(this, "Completa usuario o correo y clave", Toast.LENGTH_SHORT).show() return } val usuario = dbHelper.loginConUsuarioOCorreo(usuarioOEmail, clave) if (usuario != null) { val intent = Intent(this, DashboardActivity::class.java) intent.putExtra("nombres", usuario.nombres) startActivity(intent) finish() } else { Toast.makeText(this, "Credenciales incorrectas", Toast.LENGTH_SHORT).show() } } }

Proceso de registro

  1. Leer todos los campos del formulario.
  2. Validar vacios, longitud maxima, longitud de DNI y longitud de clave.
  3. Verificar si ya existe el mismo usuario, email o DNI.
  4. Si no hay duplicados, insertar el nuevo registro en SQLite.
  5. Volver al login con un mensaje de registro correcto.

Puntos que conviene revisar si algo falla

  • Confirmar que la tabla usuario se haya creado correctamente.
  • Verificar que no exista android.R importado por error.
  • Revisar que los ids del XML coincidan exactamente con los usados en findViewById().
  • Comprobar que el registro no repita usuario, email o DNI.
  • Recordar que el login acepta usuario o correo, pero siempre necesita la clave correcta.

PARTE 8: Propuesta final

Como se realizaria de forma ordenada

La forma mas clara de trabajar este proyecto es construir primero la tabla y el formulario de registro, porque sin usuario no tiene sentido el login. Luego se implementa la validacion con SQLite para aceptar usuario o correo. Finalmente se redirige a un dashboard sencillo para completar el flujo de una app moderna de acceso local.

Proceso Resultado esperado
Registro Se guarda un nuevo usuario cumpliendo limites y validaciones.
Login Se valida si existe un usuario con ese correo o usuario y la clave correcta.
Dashboard Se muestra una pantalla simple con gadgets estaticos y saludo por nombre.

Resultado esperado del proyecto

  1. Registrar un usuario nuevo en SQLite.
  2. Iniciar sesion con usuario o correo y clave.
  3. Ver un dashboard con compras, ventas, productos y saldo S/ 250.
  4. Tener una base clara para luego agregar cierre de sesion o sesion persistente.
