RecyclerView con SQLite en Android

Guia hibrida: primero el mapa del tema y luego la implementacion paso a paso

Nivel sugerido: intermedio

Temario General del Manual (desarrollado)

1. 🧠 Fundamentos

Concepto: Persistencia vs. Temporalidad.

La persistencia permite que los datos sobrevivan al cierre de la app. SQLite es el motor ideal para guardar datos sin internet.

💡 Ejemplo: Las variables se borran; SQLite es permanente. 🎯 Meta: Flujo local.

2. 🏗️ Arquitectura

Organización: El patrón Modelo-Vista-Adaptador.

Separamos la lógica de datos de la interfaz visual usando el patrón M-V-A para un código limpio y ordenado.

💡 Ejemplo: El Adapter dibuja, el Helper habla con SQL. 🎯 Meta: Código organizado.

3. 🗄️ El Motor (SQLite)

Estructura: Tablas, Columnas y Tipos de Datos.

Crearemos el Helper que da vida a las tablas y columnas mediante sentencias SQL estándar.

CREATE TABLE producto_categoria (id INTEGER PRIMARY KEY, nombre TEXT)

🎯 Meta: Base de datos funcional.

4. 📺 Visualización

Lectura: De la tabla a la pantalla.

Mapearemos registros de la BD (Cursor) a objetos Kotlin para poder mostrarlos en el RecyclerView.

💡 Ejemplo: cursor.getString → Objeto.nombre. 🎯 Meta: Listar datos.

5. ⚙️ CRUD Dinámico

Acciones: Crear, Editar y Borrar.

Implementación del ciclo completo (Insertar, Editar, Borrar) con refresco automático de la lista visual.

db.insert("alumnos", null, values)

🎯 Meta: Gestión total de registros.

6. 🖱️ Interacción

UX: Clicks, Diálogos y Validaciones.

Crearemos formularios amigables y diálogos de validación para que el usuario interactúe con los datos.

💡 Ejemplo: Click largo abre diálogo: "¿Deseas eliminar?". 🎯 Meta: App fácil de usar.

7. 🚀 Funciones Pro

Optimización: Búsquedas y Estados Vacíos.

Añadiremos filtros de búsqueda en tiempo real y mensajes visuales personalizados cuando la lista esté vacía.

SELECT * FROM alumnos WHERE nombre LIKE '%Ana%'

🎯 Meta: Acabado profesional.

8. 🛡️ Estabilidad

Mantenimiento: Errores y Fugas de Memoria.

Manejo de recursos para evitar que la app falle, aprendiendo a cerrar conexiones y manejar excepciones.

💡 Ejemplo: Siempre cerrar el cursor con cursor.close(). 🎯 Meta: App robusta.

PARTE 1: Lo que vas a aprender

Objetivo de la guia

En esta guia vas a conectar una lista moderna de Android con una base de datos local SQLite, para que no trabajes solo con datos estaticos, sino con informacion persistente.

Concepto central

RecyclerView muestra datos; SQLite los guarda de forma permanente dentro del dispositivo.

Resultado esperado

Una app capaz de registrar elementos, listarlos, actualizarlos y eliminarlos sin perderlos al cerrar la app.

Proyecto recomendado

Agenda de contactos, lista de productos, tareas o alumnos almacenados localmente.

PARTE 2: Arquitectura de RecyclerView con SQLite

Componentes principales

Modelo

Representa cada registro. Ejemplo: Contacto, Producto o Tarea.

SQLiteOpenHelper

Adapter

Activity o Fragment

Paso 3.2: Crear el helper ProductoCategoriaDbHelper.kt

En Android Studio: package data → New → Kotlin Class/File → ProductoCategoriaDbHelper.kt

package com.example.sqlite_a.data import android.content.Context import android.database.sqlite.SQLiteDatabase import android.database.sqlite.SQLiteOpenHelper import android.util.Log // Esta clase administra la base de datos local SQLite. // Aqui se define el nombre de la BD, su version y la tabla principal. class ProductoCategoriaDbHelper(context: Context) : SQLiteOpenHelper(context, DB_NAME, null, DB_VERSION) { companion object { // Etiqueta para identificar los mensajes en Logcat. const val TAG = "DBProductCat" // Nombre del archivo fisico de la base de datos dentro del dispositivo. const val DB_NAME = "inventario.db" // Version actual de la BD. Si en el futuro cambia la estructura, // este numero debera aumentar para ejecutar onUpgrade(). const val DB_VERSION = 1 // Nombre de la tabla y de sus columnas. const val TABLE_CATEGORIA = "producto_categoria" const val COL_CAT_ID = "id" const val COL_CAT_NOMBRE = "nombre" const val COL_CAT_DESCRIPCION = "descripcion" } override fun onCreate(db: SQLiteDatabase) { // Sentencia SQL que crea la tabla producto_categoria. val createCategoriaTable = """ CREATE TABLE $TABLE_CATEGORIA ( $COL_CAT_ID INTEGER PRIMARY KEY AUTOINCREMENT, $COL_CAT_NOMBRE TEXT NOT NULL, $COL_CAT_DESCRIPCION TEXT NOT NULL ) """.trimIndent() // .trimIndent() : Permite escribir el SQL bien ordenado dentro del código. Quita los espacios de sangría sobrantes al inicio de cada línea. // Muestra en Logcat que SQLite acaba de entrar al proceso de creacion. Log.d(TAG, "onCreate(): se esta creando la tabla $TABLE_CATEGORIA") // Ejecuta la instruccion SQL para crear la tabla. db.execSQL(createCategoriaTable) // Confirma en Logcat que la tabla fue creada correctamente. Log.d(TAG, "onCreate(): tabla $TABLE_CATEGORIA creada correctamente") } override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) { // Si la version cambia, se elimina la tabla anterior // y luego se vuelve a crear con la nueva estructura. Log.d(TAG, "onUpgrade(): version antigua = $oldVersion, version nueva = $newVersion") db.execSQL("DROP TABLE IF EXISTS $TABLE_CATEGORIA") onCreate(db) } }

La tabla no se crea solo por escribir esta clase. Para que SQLite construya realmente la base de datos y la tabla, es necesario crear una instancia del helper desde la actividad y abrir la base de datos.

package com.example.sqlite_a import android.os.Bundle import android.util.Log import androidx.activity.enableEdgeToEdge import androidx.appcompat.app.AppCompatActivity import androidx.core.view.ViewCompat import androidx.core.view.WindowInsetsCompat import com.example.sqlite_a.data.ProductoCategoriaDbHelper class MainActivity : AppCompatActivity() { override fun onCreate(savedInstanceState: Bundle?) { super.onCreate(savedInstanceState) enableEdgeToEdge() setContentView(R.layout.activity_main) ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main)) { v, insets -> val systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars()) v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom) insets } // Se crea el helper de SQLite. Log.d("DBProductCat", "MainActivity: se crea la instancia del helper") val dbHelper = ProductoCategoriaDbHelper(this) // Al abrir la base de datos, Android crea inventario.db // y ejecuta onCreate() si todavia no existe. Log.d("DBProductCat", "MainActivity: se solicita writableDatabase") val db = dbHelper.writableDatabase Log.d("DBProductCat", "MainActivity: la base de datos esta lista para usarse") // Se cierra la conexion luego de esta prueba inicial. db.close() Log.d("DBProductCat", "MainActivity: conexion cerrada") } }

Para observar estos mensajes, abre Logcat en Android Studio y filtra por la etiqueta DBProductCat.

Idea clave de este paso

  • Primero se escribe la clase helper.
  • Despues se crea el objeto ProductoCategoriaDbHelper en MainActivity.
  • Cuando se usa writableDatabase o readableDatabase, Android crea la BD y la tabla si aun no existen.
  • Los mensajes Log.d(...) permiten verificar en Logcat si el proceso realmente se ejecuto.

Que estas dejando preparado desde ahora

  • Nombre general de la base de datos: inventario.db, para que sirva luego para categorias y productos.
  • Nombre del helper: ProductoCategoriaDbHelper, porque en esta etapa solo administra la tabla producto_categoria.
  • Tabla actual: producto_categoria.
  • Tabla futura: producto, que despues podra incluir una clave foranea como id_categoria.

Decisiones tecnicas de esta primera version

  • SQLiteOpenHelper crea la BD la primera vez que ejecutas la app.
  • onCreate() solo construira por ahora la tabla producto_categoria.
  • onUpgrade() se usara cuando cambies la estructura y aumentes la version de la BD.
  • Usar constantes para tablas y columnas evita errores cuando luego agregues consultas, inserts y updates.

Paso 4.1: Agregar el metodo obtenerCategorias() en ProductoCategoriaDbHelper.kt

Este metodo consultara todos los registros de la tabla, recorrera el Cursor y devolvera una lista de objetos ProductoCategoria.

Para evitar mezclar fragmentos, lo mas practico en esta parte es reemplazar todo el contenido del archivo ProductoCategoriaDbHelper.kt por el siguiente bloque completo.

package com.example.sqlite_a.data import android.content.Context import android.database.sqlite.SQLiteDatabase import android.database.sqlite.SQLiteOpenHelper import android.util.Log import com.example.sqlite_a.model.ProductoCategoria class ProductoCategoriaDbHelper(context: Context) : SQLiteOpenHelper(context, DB_NAME, null, DB_VERSION) { companion object { const val TAG = "DBProductCat" const val DB_NAME = "inventario.db" const val DB_VERSION = 1 const val TABLE_CATEGORIA = "producto_categoria" const val COL_CAT_ID = "id" const val COL_CAT_NOMBRE = "nombre" const val COL_CAT_DESCRIPCION = "descripcion" } override fun onCreate(db: SQLiteDatabase) { val createCategoriaTable = """ CREATE TABLE $TABLE_CATEGORIA ( $COL_CAT_ID INTEGER PRIMARY KEY AUTOINCREMENT, $COL_CAT_NOMBRE TEXT NOT NULL, $COL_CAT_DESCRIPCION TEXT NOT NULL ) """.trimIndent() Log.d(TAG, "onCreate(): se esta creando la tabla $TABLE_CATEGORIA") db.execSQL(createCategoriaTable) Log.d(TAG, "onCreate(): tabla $TABLE_CATEGORIA creada correctamente") } override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) { Log.d(TAG, "onUpgrade(): version antigua = $oldVersion, version nueva = $newVersion") db.execSQL("DROP TABLE IF EXISTS $TABLE_CATEGORIA") onCreate(db) } fun obtenerCategorias(): MutableList<ProductoCategoria> { // Lista donde se guardaran temporalmente los resultados. val listaCategorias = mutableListOf<ProductoCategoria>() // Se abre la base de datos en modo lectura. val db = readableDatabase // Consulta SQL para recuperar todas las categorias ordenadas por id. val query = "SELECT * FROM $TABLE_CATEGORIA ORDER BY $COL_CAT_ID ASC" // Cursor: objeto que permite recorrer fila por fila el resultado. val cursor = db.rawQuery(query, null) Log.d(TAG, "obtenerCategorias(): total recuperado = ${cursor.count}") // moveToFirst() verifica si existe al menos un registro. if (cursor.moveToFirst()) { do { // Se leen los valores de cada columna de la fila actual. val id = cursor.getInt(cursor.getColumnIndexOrThrow(COL_CAT_ID)) val nombre = cursor.getString(cursor.getColumnIndexOrThrow(COL_CAT_NOMBRE)) val descripcion = cursor.getString(cursor.getColumnIndexOrThrow(COL_CAT_DESCRIPCION)) // Cada fila se convierte en un objeto ProductoCategoria. val categoria = ProductoCategoria(id, nombre, descripcion) // Se agrega el objeto a la lista final. listaCategorias.add(categoria) } while (cursor.moveToNext()) } // Siempre se debe cerrar el cursor y la conexion al terminar. cursor.close() db.close() return listaCategorias } }

package com.example.sqlite_a.adapter import android.view.LayoutInflater import android.view.View import android.view.ViewGroup import android.widget.TextView import androidx.recyclerview.widget.RecyclerView import com.example.sqlite_a.R import com.example.sqlite_a.model.ProductoCategoria class ProductoCategoriaAdapter( private val listaCategorias: MutableList<ProductoCategoria> ) : RecyclerView.Adapter<ProductoCategoriaAdapter.ProductoCategoriaViewHolder>() { // ViewHolder: guarda las referencias de los controles de cada fila. class ProductoCategoriaViewHolder(itemView: View) : RecyclerView.ViewHolder(itemView) { val tvNombreCategoria: TextView = itemView.findViewById(R.id.tvNombreCategoria) val tvDescripcionCategoria: TextView = itemView.findViewById(R.id.tvDescripcionCategoria) } override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): ProductoCategoriaViewHolder { // Infla el XML item_producto_categoria.xml para convertirlo en una vista real. val view = LayoutInflater.from(parent.context) .inflate(R.layout.item_producto_categoria, parent, false) return ProductoCategoriaViewHolder(view) } override fun onBindViewHolder(holder: ProductoCategoriaViewHolder, position: Int) { // Obtiene el objeto actual segun la posicion. val categoria = listaCategorias[position] // Coloca los datos del objeto en los TextView de la fila. holder.tvNombreCategoria.text = categoria.nombre holder.tvDescripcionCategoria.text = categoria.descripcion } override fun getItemCount(): Int { // Devuelve cuantas filas se deben mostrar. return listaCategorias.size } }

Proceso que se logra con esta mejora

  1. El usuario escribe en el buscador y la lista se filtra en tiempo real.
  2. Si no hay registros en la BD, aparece el mensaje No hay categorias registradas.
  3. Si la busqueda no encuentra coincidencias, aparece el mensaje No se encontraron categorias.
  4. El boton flotante permite registrar una categoria sin ocupar espacio en la parte superior.

Puntos que conviene revisar si la busqueda no funciona

  • Verificar que el id del buscador sea etBuscarCategoria.
  • Confirmar que el TextWatcher este llamando a filtrarCategorias().
  • Comprobar que la lista completa se cargue en listaCategoriasCompleta.
  • Notificar siempre al adapter con notifyDataSetChanged() despues del filtro.
  • Si el proyecto no reconoce el FloatingActionButton, revisar que el tema Material este activo.

Resultado esperado al terminar la Parte 7

  1. El boton superior habra sido reemplazado por un boton flotante.
  2. La parte superior mostrara un buscador local por nombre.
  3. La lista se filtrara conforme se escriba.
  4. El mensaje de estado servira tanto para lista vacia como para busqueda sin resultados.
